### scripts/synth_survey.py (397 lines)
CLI tool: runs a concept/copy through a weighted synthetic-audience survey via local Ollama (or hosted Claude fallback). Invoke: `python3 synth_survey.py --personas personas.md --concept "text or @file.md" [--n 1000] [--model gemma4:12b] [--endpoint http://localhost:11434] [--out report.md] [--provider auto|ollama|anthropic]`. Constants: `DEFAULT_OLLAMA_MODEL="gemma4:12b"`, `DEFAULT_ANTHROPIC_MODEL="claude-sonnet-5"`, `DEFAULT_ENDPOINT="http://localhost:11434"`, `DEFAULT_N=1000`, `NUM_CTX=16384`, `NUM_PREDICT=4000`, `TEMPERATURE=0.7`. Key functions: `parse_personas()` (L87, expects `## Segment N: Name (n=X, weight=Y)` blocks), `build_system_prompt`/`build_user_prompt` (L110-146, generates Q1-Q6 + RESONANCE score prompt), `call_ollama`/`call_anthropic`/`generate` (L149-238, curl-based, stdlib only, `think:false`), `render_report` (L249, markdown with weighted overall score table). `--provider auto` tries Ollama first, falls back to Anthropic if `ANTHROPIC_API_KEY` set. Output: markdown report + prints `OVERALL_SCORE: N.N` to stdout; exits 2 if no segment scored. No third-party deps (stdlib + curl subprocess).

### scripts/synth_usability.py (438 lines)
CLI tool: synthetic usability test — walks personas x tasks through a described UI, hunting for failure (not politeness). Invoke: `python3 synth_usability.py --personas personas.md --ui ui.md --tasks tasks.md [--model gemma4:12b] [--endpoint ...] [--out usability-report.md] [--provider auto|ollama|anthropic]`. Same model/endpoint constants as synth_survey.py (`gemma4:12b` / `claude-sonnet-5` fallback, `NUM_CTX=16384`, `NUM_PREDICT=1500`, `TEMPERATURE=0.6`). Personas file: same `## Segment N: Name (n=X, weight=Y)` format (n ignored here, weight drives scoring); warns via `warn_if_no_accessibility_persona()` (L109) if no screen-reader/low-vision persona present — Forge always wants one. Tasks file: `## Task N: Name` blocks (`parse_tasks` L96). UI file: free-form markdown/text description of the interface. Per-run output fields extracted via regex: `COMPLETED` (yes/partial/no), `CONFIDENCE` (1-10), `PROBLEMS` bullet list (`extract_problems` L238). `render_report()` (L249) builds per-persona summary table + a ranked punch list of failed/low-confidence task-runs, worst first. Prints `OVERALL_SCORE: N.N | incomplete task-runs: X/Y`; exits 2 if nothing scored.
